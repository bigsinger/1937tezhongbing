# 开发路线图

## M0：工程与格式探针（当前）

- [x] 双子项目仓库结构；
- [x] Godot 4.7 可运行占位原型；
- [x] .NET 10 GFL 枚举、中文名称解码、索引交叉校验与本地提取；
- [x] 十二个 VWF 网格/SLIST1 边界和 126 条 SLF 映射验证；
- [x] 已知原版 SHA-256 输入校验；
- [x] 合成 fixture 测试和原版资产泄漏守卫；
- [ ] 独立验证 SPR1/TLG1 的 LZO1X 与 RGB565 解码。

退出条件：工具能稳定识别原版目录，GFL 枚举固定为 1394 条，Godot 4.7 本机运行验证原型完成选择与移动。资源识别、Godot 4.7.1 全脚本解析、移动逻辑测试和主场景启动测试均已完成；人工交互与帧时间验收会在 M1 首批真实画面导入后一起执行。

## M1：第一张真实画面

- 导出一个 TLG1 地形组为 PNG；
- 导出一个角色的完整方向/动作帧；
- 导出并播放一个 WAV；
- 定义带版本号、来源记录和锚点的资源 manifest。

退出条件：Godot 场景显示一块原版地形、一个可移动角色，并播放本地导入的音效。

## M2：关卡 0 数据

- 解析 VWF 地图尺寸、图层、静态对象和角色起点；
- 解析 DBL 对象属性和 SLF 音频映射；
- 建立遮挡、碰撞、深度排序和相机边界；
- 输出规范化 `map-v1.json`。

退出条件：能在新引擎中只读漫游第一关并与原版关键坐标对齐。

## M3：战术垂直切片

- 五名队员选择、编队、寻路和姿态；
- 视野锥、听觉、潜行、警戒和尸体发现；
- 武器、弹药、伤害、背包和交互；
- 一个可完成的任务目标、失败条件、保存/读取；
- 固定输入回放、状态哈希和帧时间基线。

退出条件：第一关的一个封闭任务片段从开始到成功/失败都可玩。

## M4：完整内容复原

- 逐关实现任务脚本和剧情流程；
- 校准角色动画、AI 时序、难度和数值；
- 过场视频、字幕、菜单、教程与无障碍设置；
- 现代窗口/无边框全屏、高清 UI、按键重映射和音量分组。

## 性能目标

- 1920×1080、60 FPS；
- P95 帧时间不超过 16.7 ms，P99 不超过 25 ms；
- 正常游玩期间不出现超过 50 ms 的资源加载尖峰；
- 十分钟自动回放无状态漂移、无持续内存增长。

# 开发路线图

本路线图区分四种状态：格式可解析、资产可转换、数据可在 Godot 中运行、任务可完整游玩。前面三项不能替代最后一项。

## M0：工程与格式基线（已完成）

- [x] Patch/Remake 双子项目仓库结构；
- [x] Godot 4.7.1 与 .NET 10 工具链；
- [x] 合成 fixture 测试和意外资产收录守卫；
- [x] 1394 条 GFL 枚举、GBK 名称解码和伴随索引交叉校验；
- [x] 已知版本 SHA-256 校验；
- [x] LZO1X、RGB565、alpha 和黑色透明键；
- [x] IBLOCK、TLG1、SPR1、DBL、VWF、SLIST1 和 SLF/WAV 的严格解析；
- [x] 十二个正式 VWF/SLIST 精确解析到 EOF。

退出条件已经满足：工具可以拒绝未知版本，自动测试不依赖批量原版资产，无资产时 Godot 工程也能安全启动。

## 性能与兼容性目标

- 1920×1080、60 FPS；
- P95 帧时间不超过 16.7 ms，P99 不超过 25 ms；
- 正常游玩期间不出现超过 50 ms 的同步资源加载尖峰；
- 十分钟自动回放无状态漂移、无持续内存增长；
- 默认 Godot Compatibility renderer，可在 Windows 10/11 和常见旧集成显卡运行。

这些是最终验收目标。m000 的 44 项窗口探针曾在 360 帧样本中维持 60 FPS，并通过双人营救—全员撤离—重置的真实 scene 闭环；本地试玩包的 headless 冒烟仍只证明可启动。当前已补事件级状态哈希，但后续仍需以真实帧输入覆盖长时间交战、投射物、拾取和媒体切换，验证同步加载尖峰、内存和十分钟稳定性。

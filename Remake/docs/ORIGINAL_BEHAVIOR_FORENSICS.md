本文记录为复刻实现提供约束的原版行为结论。仓库不收录原版可执行文件、IDA 数据库、反编译日志、界面图片或其他原始素材；地址级结论来自仓库外的只读分析记录 `E:\1937\original-forensics-20260719\原版行为取证报告.md`。

原版实机取证有一项可审计副作用：原程序在 `2026-07-19 00:39:05` 更新了仓库外游戏目录中的 `E:\1937\1937tzb_1229\1937M010.SAV`（388 字节，SHA-256 `82F5B83597E1D9DB959EC92FE3D97F35E82BD9A2BB589B3DD3EDCC66DA72DADE`）。该文件未进入仓库或补丁包；本记录保留此事实，便于使用者自行恢复原目录备份。

## 如何阅读证据等级

| 标签 | 含义 |
|---|---|
| `recovered` / 原版确证 | 有原程序控制流、资源字段或原版实机行为直接支持，可以作为兼容约束 |
| `mixed` | 目标、scene 或资源引用来自原版，但呈现方式包含复刻设计 |
| `remake_editorial` | 为现代可玩性新增或补写，不能表述成原版事实 |
| `unresolved_remake_default` | 原版路径已定位，但精确数值或语义尚未恢复，暂用可玩的重制默认值 |

## 四条原版绘制队列

原版不是把所有对象简单按贴图中心或世界 Y 排序。`sub_44E2D0`（`0x44E2D0`）读取 DBL sprite 的 `header[0]` 分组，`sub_44EF50`（`0x44EF50`）按固定顺序绘制：

| `header[0]` | 原版队列 | 原版规则 | 典型资源 |
|---:|---|---|---|
| 1 | 地面/固定背景 | 最先绘制，始终位于人物和正常物件后 | 地面、道路、阴影、庄稼地 a/b |
| 0 | 正常深度队列 | 与人物一起按逐层基线稳定排序 | 人物、稻谷、树、墙、箱子 |
| 2 | 空中/固定前景 | 正常队列之后绘制，始终覆盖普通人物 | 电线、炊烟、空中旗帜、建筑遮挡片 |
| 3 | 顶层 | 最后绘制 | 海鸥 |

正常队列的恢复公式为：

```text
sort_key = current_world_y - sprite_y_offset + current_frame_layer_baseline
```

`sub_44E000`（`0x44E000`）只在左侧深度严格大于右侧时交换，因此相同深度保持原插入顺序。当前 Godot 运行时通过 `world_depth.gd` 把四队列映射到互不重叠的 CanvasItem z 区间，并把正常对象基线压缩到中间区间。

庄稼遮挡需要区分两种资源：

- `庄稼地a/b.spr` 的 `header[0] == 1`，是地面底片，不能覆盖人物；
- `稻谷.spr` 的 `header[0] == 0`，是独立竖向稻秆，和人物按脚底基线排序。人物在稻秆后方时可被局部遮挡，在前方时应盖住稻秆。

当前实现修正了把整块庄稼地按 Y 放进人物队列的问题，也不再绘制人物到移动目标之间的黄色连线。关键修复不只在渲染端：`ImportedLevelData` 现在保留并校验转换结果中的 `database_header_values`。m000 真实资源回归确认 22 个 DBL 336/337 庄稼底图为 queue 1 固定背景、70 个 DBL 335 稻谷为 queue 0 正常基线对象；完整真实资产套件持续检查这些数量与队列约束。多层 SPR 的逐层 baseline 尚需继续做像素级对照；现在的单层静态对象以导出的 `reference_y` 为近似基线。

## 复刻版默认键位与明确扩展

复刻版以原版主键位为默认值，并允许在设置中重映射。`S` 当前直接显示敌军视野与攻击范围，是 `remake_editorial` 可视化增强；`B` 当前让玩家点击阵亡敌人清理现场，是对原版目标命令的可玩解释，不宣称与原二进制逐语义相同。

为适配已经实现的现代玩法，复刻另加下列快捷键：

| 输入 | 复刻功能 | 来源 |
|---|---|---|
| `E` | 营救、拾取与任务交互 | `remake_editorial` |
| `Q` | 主动换弹 | `remake_editorial` |
| `F` | 引爆已经安放的任务炸药 | `remake_editorial` |
| `Tab` / `Shift+Tab` | 正向/反向轮换已持有武器 | `remake_editorial` |
| `Ctrl+F5` | 快速保存；避免占用原版 F5 选择第 4 名队员 | `remake_editorial` |
| `Ctrl+F9` | 按保存时间读取最新有效槽；菜单“读取”仍打开多槽选择器 | `remake_editorial` |
| `Shift+左键` | 多选增减 | `remake_editorial` |
| 左右/下方向键、中键拖动、滚轮 | 镜头平移/缩放补充操作；`W/A` 与原背包键冲突，`↑` 保留原强制目标语义 | `remake_editorial` |

`PageUp` / `PageDown` 只在调试构建中切关，不属于发行版操作。当前设置页可捕获带 Ctrl/Alt/Shift 的组合键；原版等价的 `Ctrl`/`↑` 强制目标通道也分别列出并参与按住状态重映射。若新键与已有动作冲突，两项会交换，并可一键恢复默认键位；简报、对白和影片的关闭键同步使用重映射后的系统菜单键，不再另行硬编码 `Esc`。

## 右侧五列背包/武器栏

原版 `A` 物品栏与 `W` 武器栏互斥；再次按相同键或按 `Esc` 关闭。选择项目后原版立即关闭栏位并回到世界，不支持拖拽、排序、翻页或格子互换。

原版布局由代码常量和 UI 资源共同确认：

```text
popup_x = screen_width - 276
popup_y = screen_height - 62 - 421
icon_x = popup_x + 13 + 50 * (index % 5)
icon_y = popup_y + 40 + 84 * floor(index / 5)
```

即弹窗宽 276px、右对齐、位于 62px 底部 HUD 上方，项目严格五列；横向列宽/步距 50px，格高 74px、纵向行距 84px，数量显示在格内下部。当前复刻的 `InventoryGridView` 使用同样的右侧 276×421 布局和五列 50×74 格，分别由 `A`/`W` 打开物品或武器页。视觉材质仍需逐像素贴近原版底图。

## M 键右下角地图

原版每关只加载一张已经烘焙红圈的静态图，`M` 或 50×50 HUD 按钮切换显示；它不显示队员、敌人、镜头框，也不实时更新。位置为：

```text
map_x = screen_width - map_width
map_y = screen_height - 62 - map_height
```

十二张图宽高各异，范围从 206×146 到 346×226。当前复刻保持右下角、底栏上方和 `M` 切换，不暂停战斗，并在可用时以关卡图/地形作为底图；动态绘制蓝色我方点、红色敌军点、黄色任务点及镜头框，并允许点击卷屏。这些实时点位和点击行为是有意的 `remake_editorial` 增强，不是原版恢复事实。

## 系统菜单、十槽存档与音量

原版 `Esc` 菜单的保存、读取各有 10 个槽位，后续由鼠标操作，`Esc` 或右键返回上一级。当前复刻也提供 10 个手动槽位，另保留 `quick` 快速槽和 `autosave` 自动槽；覆盖已有手动存档需再次确认。菜单中的“读取”始终打开多槽选择器，只有复刻扩展 `Ctrl+F9` 会按时间读取最新有效槽。任务失败时画面灰化并打开标题为“任务失败”的菜单，正文模板为 `任务失败：%s\n可重新开始本关，或从多槽存档选择器读取进度。`（`%s` 为失败原因）；继续和保存不可用，重玩与读取可用，失败状态也不能写入有效存档。已掩埋敌人的 scene 索引会随局内状态保存，读取后不会让尸体重新出现。

复刻设置 schema 当前包含：

- 总静音开关以及主音量、音乐、音效、语音四条独立滑杆，分别接到 `Master`、`Music`、`Sfx`、`Voice` 总线；
- 全屏跟随当前桌面分辨率、字幕、关卡简报和鼠标边缘卷屏；
- 可持久化的按键重映射和恢复默认键位。

运行时有独立音乐/环境声播放器并路由到 `Music`；影片播放器的音轨也进入 `Music`，对白进入 `Voice`，武器、爆炸、环境事件和 UI 音效进入固定 SFX 池。媒体模态层中的 `Esc` 在松开时提交关闭/跳过，避免一次按键同时穿透到下面的系统菜单；Enter/Space 用于继续简报或对白。

以上设置、十槽选择器和 JSON 原子写入属于复刻产品功能；除十槽菜单结构外，不应归因于原版。

## 十二关导演数据的证据边界

`mission_direction.json` 已为 `m000`—`m011` 提供第一版数据驱动导演：43 个节奏节点、45 行提示对白、逐关教程门控、镜头请求、AI 协作和 Easy/Normal/Hard 难度换算。其证据等级不是“一整关都已恢复”：

- 任务 ID、objective、scene binding 和部分镜头目标来自已恢复的任务控制流/关卡锚点，标 `recovered_scene_binding` 或 `mixed`；
- 对白措辞、镜头时长与缩放、教程文本、AI 策略及难度数值均为 `remake_editorial`；
- 目录明确声明 `original_dialogue_claimed=false`，验证器拒绝把补写对白伪标为原版逐字稿。

因此当前交付是“十二关可执行导演第一版”，仍需用原版逐关实机录像和复刻完整通关数据校对对白、镜头路径、触发时机、增援与难度曲线。详见 [十二关导演说明](MISSION_DIRECTION.md)。

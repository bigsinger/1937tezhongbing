# Windows 本地试玩包

本工程不会把原版或本地转换资源提交到 Git。试玩包只在开发者电脑的
`Remake/LocalBuild/1937Remake/` 中生成，该目录已由根 `.gitignore` 排除。

当前开发机的默认启动入口为：

```text
F:\bigsinger\1937tezhongbing\Remake\LocalBuild\1937Remake\Play-1937-Remake.cmd
```

它是本机生成物；源码、任务或资源转换结果发生变化后应重新构建，不要把旧 PCK 当作最新试玩版本。试玩前请打开同目录 `build-info.json` 核对 `git_commit` 与当前源码提交；本地构建被 Git 忽略，不会因拉取源码自动刷新。

## 一键生成

先完成本地资源导入，确保存在
`Remake/LocalAssets/converted/levels/m000/level.json`，然后在 `Remake` 目录运行：

```powershell
.\tools\Build-Playable.cmd
```

如果 Godot 4.7.1 不在 `PATH`，显式传入标准版控制台程序：

```powershell
.\tools\Build-Playable.cmd `
  -GodotExecutable "C:\Tools\Godot_v4.7.1-stable_win64_console.exe"
```

生成结果：

```text
LocalBuild/1937Remake/
├── Play-1937-Remake.cmd       双击启动入口
├── README.txt
├── build-info.json
├── smoke-test.log
├── executable-smoke-test.log
├── game/
│   ├── 1937Remake.exe
│   └── 1937Remake.pck
└── LocalAssets/               转换资源目录或本机目录联接
```

默认的 `Junction` 模式不会再次复制数百 MiB 转换资源，适合当前电脑快速反复构建。
生成可移动目录时使用复制模式：

```powershell
.\tools\Build-Playable.cmd -AssetMode Copy
```

复制模式只放入运行所需的 `converted/`，不会复制 `raw/`、转换测试目录或原始 GFL
提取结果。无论采用哪一种模式，生成目录都不会进入 Git。

本轮涉及输入、UI、特殊对象、十二关导演和存档 schema 的修改，必须重新运行构建脚本后再验收。构建脚本会先用 PCK 加载路径验证包，再直接启动复制/导出的 `game/1937Remake.exe`；成功至少应满足两个 smoke log 都无 Godot/GDScript 错误、十槽保存页可打开、`M` 地图红点持续刷新、`A`/`W` 为五列方格、设置页显示总静音、四条音量滑杆和按键列表。

## 运行与切关

双击 `Play-1937-Remake.cmd` 启动。不要只移动 `game/1937Remake.exe`，因为当前运行时按
`game/` 的上一级寻找 `LocalAssets/converted/`；启动器也会固定进程工作目录，保证 PCK
运行方式下该相对路径稳定。

命令行可直接选择关卡：

```powershell
.\LocalBuild\1937Remake\Play-1937-Remake.cmd -- --level=m007
```

可用关卡 ID 为 `m000`—`m011`。`PageUp` / `PageDown` 只在调试构建中切关；`R` 已按原版改为跑步/行走切换，不再重置关卡。
自动探针或不希望显示简报时可追加 `--skip-briefing`。

| 直接选择已持有项目 | `1`—`0`：匕首、弹弓、大刀、飞刀、手枪、步枪、机枪、地雷、手榴弹、炸药包 |
| 轮换已持有武器 | `Tab`；`Shift + Tab` 反向 |
| 关闭简报/下一句，或跳过媒体 | `Enter` / `Space` 继续；松开 `Esc` 跳过（模态媒体期间任务计时和战斗暂停） |
| 平移镜头 | 鼠标移到窗口最外侧 1 像素；左右/下方向键或中键拖动是复刻扩展，原版 `↑` 保留为强制目标按住键 |
| 缩放 | 鼠标滚轮 |

`S`/`B` 的原版帮助图标签分别是“视线观察模式”和“掩埋模式”，但地址级运行路径进一步表明：S 会把下一次存活友军左击变为单选/观察目标，B 会进入尸体或失能友军目标命令。当前复刻的范围显示和阵亡敌人清理是明确的增强/可玩解释，并未声称逐语义复现。完整原版键表、F8/F9 调试残留和鼠标证据见[原版行为取证摘要](ORIGINAL_BEHAVIOR_FORENSICS.md)。所有表中可重映射的键都可在“设置 → 按键设置”修改；发生冲突时两个动作交换绑定，也可以恢复默认值。

界面右侧显示任务进度，左上显示所选队员的武器、弹匣/备弹、生命和地雷。`A`/`W` 弹出的背包按原版运行代码采用 276×421、右侧对齐的五列方格，数量和当前装备状态直接显示在格内。出口每 0.2 秒自动检查 128 像素范围；默认要求全部存活队员和本关护送角色，存在 `exit_party` 规则时按剧情指定成员判定，例如 m001 只要求古明与铁蛋爹、m002 要求老赵与获救的强子。m010 不使用 `E`：老赵、强子、大牛、古明必须分别同时处在四个 128 像素制高点内，系统自动判定；按 `M` 打开地图时，四个占点位置会显示为任务标记。

原版 M 地图只是右下角的逐关静态烘焙图，不含动态红点。复刻保持右下角和底栏上方的位置关系，同时增加实时我方/敌方/任务点、镜头框和点击卷屏，属于有意的现代可玩性增强；打开地图不会暂停单位移动和 AI。

爆破点通常按 `E` 交互，但不是所有关卡都会扣除背包炸药。m001、m004、m011 的目标按预置炸药/引燃物处理；m002、m003、m008、m009 每个目标需要一份 DBL 998 炸药，数量不足时界面提示且任务不推进。炸药只在任务运行时接受当前 scene 事件后扣除，重复或无效交互不会丢失物品。m008 是特殊闭环：先按目标要求布置，再按 `F` 手动引爆；通用 `E` 不能完成最后的引爆步骤。

## 游戏菜单、设置与存档

正常游戏时松开 `Esc` 打开暂停菜单。菜单暂停任务计时、AI 和战斗，可继续游戏、打开多槽存档读取器、保存、重新开始本关或退出；菜单内松开鼠标右键也可返回上一级。任务失败后，游戏画面会灰化并自动打开标题为“任务失败”的菜单，正文模板为 `任务失败：%s\n可重新开始本关，或从多槽存档选择器读取进度。`（`%s` 为失败原因）；继续和保存不可用，重玩本关和从任一有效槽读取可用，且失败状态不能覆盖已有有效存档。`R` 在游戏世界中始终是跑/走切换，不承担重开功能。

菜单中的设置会立即应用并持久保存：

- 全屏使用 Windows 当前桌面分辨率，不把 1280×720 项目设计尺寸当作独占显示分辨率；
- 主音量、音乐、音效和语音四条滑杆，分别控制 `Master`、`Music`、`Sfx`、`Voice` 音频总线；
- 可捕获 Ctrl/Alt/Shift 组合键的按键重映射，以及一键恢复默认键位；
- 语音字幕；
- 切换关卡时是否显示任务简报；
- 鼠标移动到窗口边缘时是否卷屏。

保存/读取菜单提供 `slot_1`—`slot_10` 十个手动槽；覆盖已占用槽需再次点击确认。读取页还会列出存在的 `quick` 和 `autosave`，`Ctrl+F5` 写入 `quick`，`Ctrl+F9` 读取时间最新的有效存档，任务胜利时静默更新 `autosave`。任务失败状态不能覆盖有效存档。存档覆盖当前局内可变状态：

- 当前关卡、任务耗时、目标完成/计数/去重值、失败状态和持久任务事实；
- 队员、敌人和护送角色的位置、阵营、生命/死亡、匍匐/选择状态，玩家背包、武器、弹匣与备弹；
- 敌人巡逻/搜索状态，护送角色的获救与跟随关系；
- 已激活任务 scene、关卡公共物品、尚未拾取的地图物品、汽油桶剩余生命或已销毁状态、尚未取得的任务掉落；
- 已部署地雷、type 8/10 世界对象及 type 11 AI 控制状态、尚未结算的在途投射物；
- 已掩埋敌人的 scene 索引；读取后这些尸体保持隐藏且不重新参与处理；
- 十二关导演已触发节拍、教程门控/持久事件/计时，以及 AI 姿态、增援预算和命令序号；
- 镜头位置/缩放，以及已完成关卡和最高解锁关卡的战役进度。

静态地形、导航网格、纹理、节点引用和信号连接不会写入存档；读取时先从本地转换资源重建对应关卡，再恢复上述可变状态。未完成的鼠标移动/攻击命令、当前动画的精确帧和打开中的菜单/媒体层也不序列化，角色会从安全状态继续；已经离开发射者的世界投射物会恢复。

设置和存档均为带 `schema_version` 的 JSON。写入过程先在同目录生成临时文件、重新读取并校验，再替换主文件；上一个有效版本保留为 `.bak`。主文件缺失或损坏时会读取备份；下一次写入会先把损坏主文件隔离为 `.corrupt`，不会反过来覆盖已知有效备份。

## 导出模板与回退方式

`game/export_presets.cfg` 提供 `Windows Desktop` 预设。构建脚本会检查
`%APPDATA%\Godot\export_templates\4.7.1.stable\windows_release_x86_64.exe`：

- 已安装匹配模板：生成标准 Godot release 导出；
- 未安装模板：导出相同内容的 PCK，并复制当前 Godot 4.7.1 标准版作为本地运行器。

后一种方式无需为了 Windows 本机试玩下载约 1.28 GB 的全平台模板包，生成目录仍可直接
运行；它只用于本地且不会提交。两种方式最后都会从生成目录启动 8 个 headless 帧，日志中
只要出现 Godot 或 GDScript 错误，构建即失败。

`build-info.json` 会记录 Git 提交、Godot 版本、构建方式和资产模式，便于确认反馈对应的源码版本。构建脚本只允许输出到 `Remake/LocalBuild/` 的子目录，并在替换旧输出前拒绝删除重解析点，避免误删目录联接目标。

# Windows 本地试玩包

本工程不会把原版或本地转换资源提交到 Git。试玩包只在开发者电脑的
`Remake/LocalBuild/1937Remake/` 中生成，该目录已由根 `.gitignore` 排除。

当前开发机的默认启动入口为：

```text
F:\bigsinger\1937tezhongbing\Remake\LocalBuild\1937Remake\Play-1937-Remake.cmd
```

它是本机生成物；源码、任务或资源转换结果发生变化后应重新构建，不要把旧 PCK 当作最新试玩版本。

## 一键生成

先完成本地资源导入，确保存在
`Remake/LocalAssets/converted/levels/m000/level.json`，然后在 `Remake` 目录运行：

```powershell
.\tools\Build-Playable.cmd
```

如果 Godot 4.7.1 不在 `PATH`，显式传入标准版控制台程序：

```powershell
.\tools\Build-Playable.cmd `
  -GodotExecutable "C:\Tools\Godot_v4.7.1-stable_win64_console.exe"
```

生成结果：

```text
LocalBuild/1937Remake/
├── Play-1937-Remake.cmd       双击启动入口
├── README.txt
├── build-info.json
├── smoke-test.log
├── game/
│   ├── 1937Remake.exe
│   └── 1937Remake.pck
└── LocalAssets/               转换资源目录或本机目录联接
```

默认的 `Junction` 模式不会再次复制数百 MiB 转换资源，适合当前电脑快速反复构建。
生成可移动目录时使用复制模式：

```powershell
.\tools\Build-Playable.cmd -AssetMode Copy
```

复制模式只放入运行所需的 `converted/`，不会复制 `raw/`、转换测试目录或原始 GFL
提取结果。无论采用哪一种模式，生成目录都不会进入 Git。

## 运行与切关

双击 `Play-1937-Remake.cmd` 启动。不要只移动 `game/1937Remake.exe`，因为当前运行时按
`game/` 的上一级寻找 `LocalAssets/converted/`；启动器也会固定进程工作目录，保证 PCK
运行方式下该相对路径稳定。

命令行可直接选择关卡：

```powershell
.\LocalBuild\1937Remake\Play-1937-Remake.cmd -- --level=m007
```

可用关卡 ID 为 `m000`—`m011`。游戏内也可以按 `PageUp` / `PageDown` 切关，按 `R` 重置当前关卡。
自动探针或不希望显示简报时可追加 `--skip-briefing`。

## 当前操作

| 操作 | 输入 |
|---|---|
| 选择单个队员 | 左键 |
| 追加/取消选择 | `Shift` + 左键 |
| 编队移动 | 右键地面 |
| 攻击 | 先选队员，再右键敌人或汽油桶 |
| 游戏菜单 | `Esc`（简报/对白/视频打开时仍优先用于跳过当前媒体） |
| 战术地图 | `M`；单击地图可把镜头移到对应位置，`M` / `Esc` 返回 |
| 小队背包 | `B` 或 `I`；可查看全队状态、切换武器和为所选队员换弹 |
| 快速保存 / 读取最近存档 | `F5` / `F9`（菜单按钮使用同一套存档） |
| 营救、真实物品拾取、任务掉落和爆破点 | 靠近后按 `E` |
| m008 手动引爆 | `F`；该关不能用通用 `E` 爆破交互代替，提前引爆会失败 |
| 主动装填 | 选中队员后按 `Q` |
| 直接选择已持有武器 | `1`—`8`（依次为手枪、步枪、机枪、匕首、大刀、飞镖、弹弓、手榴弹） |
| 轮换已持有武器 | `Tab`；`Shift + Tab` 反向 |
| 在所选队员脚下布雷 | `X`（消耗地雷物品 43） |
| 关闭简报/下一句，或跳过媒体 | `Enter` / `Space`；`Esc` 跳过（模态媒体期间任务计时和战斗暂停） |
| 平移镜头 | `WASD` / 方向键、按住鼠标中键拖动，或把鼠标移到屏幕边缘 |
| 缩放 | 鼠标滚轮 |
| 重玩当前关 | `R` |

界面右侧显示任务进度，左上显示所选队员的武器、弹匣/备弹、生命和地雷。出口每 0.2 秒自动检查 128 像素范围；默认要求全部存活队员和本关护送角色，存在 `exit_party` 规则时按剧情指定成员判定，例如 m001 只要求古明与铁蛋爹、m002 要求老赵与获救的强子。m010 不使用 `E`：老赵、强子、大牛、古明必须分别同时处在四个 128 像素制高点内，系统自动判定；按 `M` 打开地图时，四个占点位置会显示为蓝色制高点任务标记。

爆破点通常按 `E` 交互，但不是所有关卡都会扣除背包炸药。m001、m004、m011 的目标按预置炸药/引燃物处理；m002、m003、m008、m009 每个目标需要一份 DBL 998 炸药，数量不足时界面提示且任务不推进。炸药只在任务运行时接受当前 scene 事件后扣除，重复或无效交互不会丢失物品。m008 是特殊闭环：先按目标要求布置，再按 `F` 手动引爆；通用 `E` 不能完成最后的引爆步骤。

## 游戏菜单、设置与存档

正常游戏时按 `Esc` 打开暂停菜单。菜单暂停任务计时、AI 和战斗，可继续游戏、保存、读取最近存档、重新开始本关或退出。任务失败后，游戏画面会灰化并自动打开不可继续的失败菜单；此时可重玩本关或读取最近存档，且失败状态不能覆盖已有有效存档。

菜单中的设置会立即应用并持久保存：

- 全屏使用 Windows 当前桌面分辨率，不把 1280×720 项目设计尺寸当作独占显示分辨率；
- 主音量；
- 语音字幕；
- 切换关卡时是否显示任务简报；
- 鼠标移动到窗口边缘时是否卷屏。

`F5` 写入快速存档 `quick`，`F9` 读取时间最新的有效存档；菜单按钮调用同一接口。任务胜利时还会静默更新 `autosave`。存档覆盖当前局内可变状态：

- 当前关卡、任务耗时、目标完成/计数/去重值、失败状态和持久任务事实；
- 队员、敌人和护送角色的位置、阵营、生命/死亡、匍匐/选择状态，玩家背包、武器、弹匣与备弹；
- 敌人巡逻/搜索状态，护送角色的获救与跟随关系；
- 已激活任务 scene、关卡公共物品、尚未拾取的地图物品、汽油桶剩余生命或已销毁状态、尚未取得的任务掉落；
- 已部署地雷及其计时状态、尚未结算的在途投射物；
- 镜头位置/缩放，以及已完成关卡和最高解锁关卡的战役进度。

静态地形、导航网格、纹理、节点引用和信号连接不会写入存档；读取时先从本地转换资源重建对应关卡，再恢复上述可变状态。未完成的鼠标移动/攻击命令、当前动画的精确帧和打开中的菜单/媒体层也不序列化，角色会从安全状态继续；已经离开发射者的世界投射物会恢复。

设置和存档均为带 `schema_version` 的 JSON。写入过程先在同目录生成临时文件、重新读取并校验，再替换主文件；上一个有效版本保留为 `.bak`。主文件缺失或损坏时会读取备份；下一次写入会先把损坏主文件隔离为 `.corrupt`，不会反过来覆盖已知有效备份。

## 导出模板与回退方式

`game/export_presets.cfg` 提供 `Windows Desktop` 预设。构建脚本会检查
`%APPDATA%\Godot\export_templates\4.7.1.stable\windows_release_x86_64.exe`：

- 已安装匹配模板：生成标准 Godot release 导出；
- 未安装模板：导出相同内容的 PCK，并复制当前 Godot 4.7.1 标准版作为本地运行器。

后一种方式无需为了 Windows 本机试玩下载约 1.28 GB 的全平台模板包，生成目录仍可直接
运行；它只用于本地且不会提交。两种方式最后都会从生成目录启动 8 个 headless 帧，日志中
只要出现 Godot 或 GDScript 错误，构建即失败。

`build-info.json` 会记录 Git 提交、Godot 版本、构建方式和资产模式，便于确认反馈对应的源码版本。构建脚本只允许输出到 `Remake/LocalBuild/` 的子目录，并在替换旧输出前拒绝删除重解析点，避免误删目录联接目标。

# Windows 本地试玩包

本工程不会把原版或本地转换资源提交到 Git。试玩包只在开发者电脑的
`Remake/LocalBuild/1937Remake/` 中生成，该目录已由根 `.gitignore` 排除。

当前开发机的默认启动入口为：

```text
F:\bigsinger\1937tezhongbing\Remake\LocalBuild\1937Remake\Play-1937-Remake.cmd
```

它是本机生成物；源码、任务或资源转换结果发生变化后应重新构建，不要把旧 PCK 当作最新试玩版本。

## 一键生成

先完成本地资源导入，确保存在
`Remake/LocalAssets/converted/levels/m000/level.json`，然后在 `Remake` 目录运行：

```powershell
.\tools\Build-Playable.cmd
```

如果 Godot 4.7.1 不在 `PATH`，显式传入标准版控制台程序：

```powershell
.\tools\Build-Playable.cmd `
  -GodotExecutable "C:\Tools\Godot_v4.7.1-stable_win64_console.exe"
```

生成结果：

```text
LocalBuild/1937Remake/
├── Play-1937-Remake.cmd       双击启动入口
├── README.txt
├── build-info.json
├── smoke-test.log
├── game/
│   ├── 1937Remake.exe
│   └── 1937Remake.pck
└── LocalAssets/               转换资源目录或本机目录联接
```

默认的 `Junction` 模式不会再次复制数百 MiB 转换资源，适合当前电脑快速反复构建。
生成可移动目录时使用复制模式：

```powershell
.\tools\Build-Playable.cmd -AssetMode Copy
```

复制模式只放入运行所需的 `converted/`，不会复制 `raw/`、转换测试目录或原始 GFL
提取结果。无论采用哪一种模式，生成目录都不会进入 Git。

## 运行与切关

双击 `Play-1937-Remake.cmd` 启动。不要只移动 `game/1937Remake.exe`，因为当前运行时按
`game/` 的上一级寻找 `LocalAssets/converted/`；启动器也会固定进程工作目录，保证 PCK
运行方式下该相对路径稳定。

命令行可直接选择关卡：

```powershell
.\LocalBuild\1937Remake\Play-1937-Remake.cmd -- --level=m007
```

可用关卡 ID 为 `m000`—`m011`。游戏内也可以按 `PageUp` / `PageDown` 切关，按 `R` 重置当前关卡。
自动探针或不希望显示简报时可追加 `--skip-briefing`。

## 当前操作

| 操作 | 输入 |
|---|---|
| 选择单个队员 | 左键 |
| 追加/取消选择 | `Shift` + 左键 |
| 编队移动 | 右键地面 |
| 攻击 | 先选队员，再右键敌人或汽油桶 |
| 营救、真实物品拾取、任务掉落和爆破点 | 靠近后按 `E` |
| m008 手动引爆（提前引爆会失败） | `F` |
| 主动装填 | 选中队员后按 `Q` |
| 直接选择已持有武器 | `1`—`8`（依次为手枪、步枪、机枪、匕首、大刀、飞镖、弹弓、手榴弹） |
| 轮换已持有武器 | `Tab`；`Shift + Tab` 反向 |
| 在所选队员脚下布雷 | `X`（消耗地雷物品 43） |
| 关闭简报/下一句，或跳过媒体 | `Enter` / `Space`；`Esc` 跳过（模态媒体期间任务计时和战斗暂停） |
| 平移镜头 | `WASD` 或按住鼠标中键拖动 |
| 缩放 | 鼠标滚轮 |
| 重玩当前关 | `R` |

界面右侧显示任务进度，左上显示所选队员的武器、弹匣/备弹、生命和地雷。出口每 0.2 秒自动检查 128 像素范围；默认要求全部存活队员和本关护送角色，存在 `exit_party` 规则时按剧情指定成员判定，例如 m001 只要求古明与铁蛋爹、m002 要求老赵与获救的强子。m010 不使用 `E`：老赵、强子、大牛、古明必须分别同时处在四个 128 像素制高点内，系统自动判定。

爆破点同样按 `E` 交互，但不是所有关卡都会扣除背包炸药。m001、m004、m011 的目标按预置炸药/引燃物处理；m002、m003、m008、m009 每个目标需要一份 DBL 998 炸药，数量不足时界面提示且任务不推进。炸药只在任务运行时接受当前 scene 事件后扣除，重复或无效交互不会丢失物品。

## 本轮试玩范围与已知边界

当前包已经包含实际伤害、玩家背包与武器切换、有限弹药/装填、攻击动作末帧命中、飞镖/弹弓/手榴弹世界投射物、受伤/死亡、攻击/友军死亡警报、真实关卡拾取、地雷、可爆汽油桶和任务角色掉落，以及救援—击毙—拾取—爆破/占点—撤离—胜负事件链。导入本地资产时，切关会显示原简报图，并通过一条独立语音/对白通道和固定 8 路音效池播放已分类的原攻击、命中、警报、角色、死亡和 UI WAV；缺失时安全降级。m000 在简报关闭后给出重制教程提示，营救彭鑫后播放老赵确认音；m006 接头点完成后显示重制提示，m011 胜利后显示恢复映射的结局图。`m000` 仍是当前演练最充分的端到端流程。

以下内容仍属于开发中，不应据此判断最终品质：

- 受伤使用 0.18 秒闪红/硬直，没有冒充尚未找到的原版独立受伤动画；
- 敌人暂为无限弹药；弹匣、备弹、装填时间和玩家攻击恢复时间是重制默认值；
- type 6/7/9 已生成世界投射物，但速度、弧高、碰撞半径、手榴弹延时/爆炸椭圆/敌我伤害仍是重制默认；type 8/10/11 专用对象尚未恢复；
- 真实拾取实体身份和弹药物品映射已恢复，但拾取数量、治疗值、地雷/油桶生命、延时、范围和伤害仍是重制默认；
- m004 已确认 scene 2637 携带物品 101；m009 默认修复为两份文件、全关清敌和四个列车爆破；m010 的四名指定队员同时占点规则已恢复。m005/m006 名称差异和其他逐关细节仍需校准；
- 原 WAV 与简报已经接线，任务数据也支持开场/目标/剧情锚点/胜利媒体 cue；目前只有上述四个触发点，编辑提示均有来源标签，没有夸称已恢复十二关完整对白或镜头。视频需先转码，播放/跳过接口尚未编排进逐关流程；
- 存档、菜单/按键重映射、逐关镜头、完整教程、AI 配合、难度和十分钟真实输入回放尚未完成。

反馈问题时请同时提供 `build-info.json`、关卡 ID、复现步骤和截图；如果是启动失败，再附上 `smoke-test.log`。

## 导出模板与回退方式

`game/export_presets.cfg` 提供 `Windows Desktop` 预设。构建脚本会检查
`%APPDATA%\Godot\export_templates\4.7.1.stable\windows_release_x86_64.exe`：

- 已安装匹配模板：生成标准 Godot release 导出；
- 未安装模板：导出相同内容的 PCK，并复制当前 Godot 4.7.1 标准版作为本地运行器。

后一种方式无需为了 Windows 本机试玩下载约 1.28 GB 的全平台模板包，生成目录仍可直接
运行；它只用于本地且不会提交。两种方式最后都会从生成目录启动 8 个 headless 帧，日志中
只要出现 Godot 或 GDScript 错误，构建即失败。

`build-info.json` 会记录 Git 提交、Godot 版本、构建方式和资产模式，便于确认反馈对应的源码版本。构建脚本只允许输出到 `Remake/LocalBuild/` 的子目录，并在替换旧输出前拒绝删除重解析点，避免误删目录联接目标。

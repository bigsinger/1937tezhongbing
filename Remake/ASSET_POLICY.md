# 原版资产政策

## 基本原则

本仓库只保存新编写的源码、格式说明、合成测试数据和可自由分发的占位内容，不保存原版商业发行内容。

下列内容不得提交、上传到 GitHub Release、放入 CI artifact 或打进公开构建：

- `M1937.exe`、安装程序和原版 DLL；
- `GFL/VWF/SVT/SAV/DBL/SLF/SI0` 文件；
- 从这些文件中提取或转换得到的原版图片、精灵、地图、音频、视频与文本；
- 原版光盘镜像、绿色版压缩包和存档。

## 本地导入模型

使用者自行选择合法持有的原版目录。导入器读取该目录，将结果写入 `Remake/LocalAssets/`。这个目录及常见原版扩展名已被根 `.gitignore` 排除。

导入器若检测到输出位于任意 Git 工作树中，会调用 `git check-ignore` 验证整个输出目录；未被忽略的位置会在写入任何文件前被拒绝。输出到工作树之外时不受此限制。CI 的资产守卫在每次 push 和 pull request 上扫描整个仓库，并检查扩展名、原引擎签名、导入目录结构和 ZIP 内容。

游戏工程最终只读取规范化中间格式：

```text
LocalAssets/
  manifest.json
  images/*.png
  audio/*.{wav,ogg}
  maps/*.json
  raw/gfl/*              仅供本地格式研究
```

导入器不会把原版资产上传到网络，也不会绕过 DRM 或访问控制。当前绿色版没有观察到 DRM，但这不改变原始内容的版权状态。

## 可以进入仓库的资产

- 项目成员原创并明确允许收录的内容；
- 公共领域内容；
- 许可证允许再分发且完整保留署名、许可证和来源记录的内容；
- 为格式测试人工生成、无法还原原作品的小型二进制 fixture。

获得原权利人授权后，应把授权范围、地域、期限、可修改性和再分发条件记录在仓库中，再决定哪些资产可以进入公开构建。

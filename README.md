# 《1937特种兵：敌后武工队》保存与复刻项目

本仓库包含两个彼此独立的子项目：

- [Patch](Patch/)：让 2001 年原版在 Windows 10/11 上流畅运行的兼容补丁、源码、文档和测试资料；
- [Remake](Remake/)：使用 Godot 4.7 与 .NET 10 从零实现的现代复刻工程。

## 快速开始

需要 .NET 10 SDK、Godot 4.7.1 Standard，以及一份与已知版本匹配的原版目录：

```powershell
cd .\Remake
.\tools\Import-OriginalAssets.cmd "E:\1937\1937tzb_1229"
godot --path .\game
```

导入后松开 `Esc` 打开菜单，`M` 切换右下角实时地图，`A`/`W` 打开物品/武器五列方格，`F2`—`F6` 选择本关第 1—5 名队员；`E` 交互，`Q` 装填，`1`—`0`/`Tab` 选择已持有项目，默认按住左 `Ctrl` 或 `↑` 再左击可强制目标。`Ctrl+F5`/`Ctrl+F9` 快速保存/读取，菜单存读档使用多槽选择器；m008 的炸药必须按 `F` 手动引爆。`PageUp` / `PageDown` 只在调试构建中切关。也可直接指定关卡：

```powershell
godot --path .\game -- --level=m011
```

不导入原版资源也可以启动，工程会显示程序化占位场景。完整的构建、导入和验证命令见 [Remake 使用说明](Remake/README.md)。

## 资产收录策略

仓库不直接收录体积较大的原版可执行文件和美术、音频、地图资源，而是提供可重复运行的本地导入器。转换产物保存在被 Git 忽略的 `Remake/LocalAssets/` 中，这样可以控制仓库与 Release 体积，并让每次导入都有来源哈希、清单和一致性检查。

这是一项仓库体积与可复现构建策略，不对任何素材的权属、授权或可分发状态作法律结论。公开发布时应由发布者根据自己的素材来源与发布范围作出判断。详见 [资产收录与本地导入策略](Remake/ASSET_POLICY.md)。
